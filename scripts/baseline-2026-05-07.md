### Key observations

- WOS alone is **18.786 tokens** (28% of all commands combined).
- Shared blocks total **1.512 tokens** but get **inlined into 33 commands** by `sync-shared-blocks.sh` — actual repo footprint is ~33× higher than the source.
- Largest WOS section is **'Command roles'** at 4.216 tokens (single section bigger than several commands combined).
- Min cacheable size for Opus 4.7 is 4.096 tokens. WOS+shared = 20.298 tokens, well above the threshold (cache works).
- After P1 (WOS_CORE ≤3k), static portion drops to 4.512 tokens — still above the 4.096-token cache floor, so we keep cache benefits.
- Scenario A → C delta: **43% fewer token-equivalents** in a typical 5-step session.
- Scenario A → E (full Skills migration) delta: **40% fewer token-equivalents** — biggest win comes from lazy body load.
- Scenario A vs B (no cache fallback): cache currently saves **53%** vs cold reads. If a user clears chat per command, they pay full price every time.
