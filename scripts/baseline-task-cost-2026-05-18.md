Generated by `python3 scripts/measure-task-cost.py` (ADR-0020).

Method: chars / 4 (approximation; ~10% precision vs Claude tokenizer per ADR-0013). Cache assumptions: Anthropic 5-minute TTL; first phase writes (1.25x), subsequent phases read (0.1x). Real per-tool cache behavior may differ; this is a simulation, not a measurement (ADR-0020).

## Assumptions (locked at slice 09)

- WOS bootstrap tokens: measured from `WORKFLOW_OPERATING_SYSTEM.md` -> 17.620 tokens.
- Command file tokens: measured per phase from `commands/<name>.md`.
- User input per phase: 200 tokens (constant; typical slash-command invocation).
- TASK_STATE initial size (post-task-init): 1.500 tokens.
- TASK_STATE growth per phase: +300 tokens (accumulation across slices).
- Cache write multiplier (Anthropic 5-min TTL): 1.25x.
- Cache read multiplier (Anthropic 5-min TTL): 0.1x.

## What this is NOT

Per ADR-0020:
- Not a real model API measurement (no tool call; no API key).
- Not per-tool cache-hit instrumentation (Cursor/Claude Code/Codex etc. cache differently; out of scope per ADR-0005 multi-tool neutrality).
- Not a trend dashboard (single-baseline snapshot; consume `--json` for trend tooling).
- Not latency (latency depends on model AND tool AND network; not derivable from tokens alone).

Future deltas: re-run after structural changes; compare against this baseline by date in filename (`scripts/baseline-task-cost-<YYYY-MM-DD>.md`).
